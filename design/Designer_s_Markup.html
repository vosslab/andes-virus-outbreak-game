§03

Verdicts

Q1-Q8 * with specs
Q1 Three unreachable rooms Adjust
Add the three missing doors. None of these is intentionally isolated - the starboard promenade is just a missing mirror, and a ship where passengers can't ever reach the sun deck or the helideck flattens the playable space too much.
# data/ship.yaml - doors: - { x: 4, y: 16, dir: v } # obs_s <-> cab_s1 - { x: 31, y: 1, dir: h } # sun_deck <-> crew_q - { x: 31, y: 18, dir: h } # tender_bay <-> helideck
Three lines of YAML; cost is trivial. Helideck specifically is needed for Q6's evac goal.
Q2 Doorway density / chokepoints Adjust
Two-tier answer. Add a secondary door to every cabin run (18 occupants : 1 door is unrealistic and floods the corridor on muster). Keep atrium <-> dining, atrium <-> galley, and lifeboat exits narrow on purpose - those are the pedagogical chokepoints.
# add a 2nd door at the far end of each cabin block: - { x: 4, y: 5, dir: h } # cab_p1 mid -> corridor (in addition to x:7) - { x: 16, y: 5, dir: h } # cab_p2 mid - { x: 21, y: 5, dir: h } # cab_p3 mid # ...mirror on starboard at y:15
Goal: cabin -> corridor egress under ~6s at N=1000. Chokepoint education stays at the atrium tier.
Q3 Atrium scale Adjust
Keep the full-beam atrium - it's the visual hero of the ship and the dramatic transmission cell the educational story turns on. Add a service stairwell at R10-11 * P-Q that crew can route through, so officer/crew agents have a bypass and the player can observe crew-as-vector independently.
# new room type: service_stair: { fill: "#7c8c99", ink: "#f3e6c8", label: "Stairs" } # new room (between atrium and inner rooms, 1-tile): - { id: stair_mid, type: service_stair, x: 15, y: 10, w: 1, h: 1, crew_only: true }
Splits the atrium into public-spread (full-beam) and crew-bypass (1 tile). Gameplay & realism both improve.
Q4 Cabin density / sub-rooms Park
Keep the one-block-per-run abstraction in v1. Sub-dividing means doubling the door count and re-doing every cabin label - too much geometry churn ahead of large_crowd launch. Revisit if cabin-internal contact tracing becomes a stated educational goal.
# revisit AFTER large_crowd ships # proposed v2: each cab_pN -> 3 sub-cells, doors to corridor only # cost: ~30 new doors, ~24 new rooms - schedule with a v2 design pass
Abstraction is honest at the educational level we're targeting. Don't gold-plate before perf is fixed.
Q5 Glyphs in room labels Adjust
Yes - wire the glyphs. Render at 50% opacity, top-right corner of each room. Improves scannability in the live sim when text labels overlap with agent dots. Glyph set below; HTML entities (numeric form) carry through to SVG cleanly.
# types in data/ship.yaml - glyph field: command: { glyph: "⚓" } # U+2693 suite: { glyph: "★" } # U+2605 medical: { glyph: "✚" } # U+271A atrium: { glyph: "✦" } # U+2726 food: { glyph: "◍" } # U+25CD (was empty) wellness: { glyph: "❀" } # U+2740 (was empty) crew: { glyph: "⚙" } # U+2699 (was empty) emergency: { glyph: "⛑" } # U+26D1 (was empty) transit: { glyph: "↦" } # U+21A6 (was empty) leisure: { glyph: "♦" } # U+2666 (was empty)
Color carries type; glyph carries category at distance. Two-channel encoding survives small render sizes.
Q6 Helideck & lifeboat semantics Adjust
They should be evacuation goals, not display-only. Cruise outbreaks turn on muster drills and medical evac - these are the highest-signal events the sim can produce. Helideck = medical-evac override for severe agents; lifeboats = muster points in a new drill scenario.
# role hints (consumed by routing override): helideck: { gameplay: med_evac_goal, trigger: "agent.state == infected && severity >= 0.7" } lifeboat_p / _s: { gameplay: muster_point, trigger: "scenario == 'drill'" } # scenario to add in src/scenarios.ts: 'drill'
Adds two narrative beats (med-evac, muster) without new geometry. Both are stress tests for the gate code.
Q7 Room-type palette Adjust
One change: split medical off isolation. Reserve red-pink #f08a8a for active medical (infirmary, future clinic), and give isolation a new purple #b6a0d0 that matches the isolated agent-state color. Keeps the visual grammar of state-of-the-agent = state-of-the-room they belong in.
medical: { fill: "#f08a8a", ink: "#5a1010" } # unchanged isolation: { fill: "#b6a0d0", ink: "#3a1040", # NEW (was medical) glyph: "⊘" } # U+2298 # rooms[].type for "isolation" room -> "isolation" (new type)
Two rooms shared one color and one of them was a gameplay state. Disambiguate.
Q8 Initial agent placement Adjust
Narrative placement - let the opening tableau tell a small story the player can read in 10 seconds. Patient-zero is a casino weekend exposure; the crew galley operator is the second-chain vector; the doctor is staffing the infirmary; the kids club seeds a family-spread thread. The remaining 11 stay roughly where they are.
# data/ship.yaml - agents (deltas only): - { id: A12, name: "Carl Brandt", state: exposed, x: 23, y: 7 } # -> Casino - { id: A06, name: "Omar Haddad", state: crew, x: 19, y: 7 } # -> Galley - { id: A07, name: "Inez Cruz", state: crew, x: 5, y: 12 } # -> Infirmary - { id: A09, name: "Sora Matsui", state: healthy, x: 28, y: 12 } # -> Kids Club - { id: A03, name: "Dre Okafor", state: symptomatic, x: 9, y: 12 } # -> Isolation
Costs nothing at runtime; gains a readable opening. Symptomatic agent already in isolation seeds the "early-detection worked" reading.
§04

Three things you didn't ask for

off-list * own them or push back
X1 Tile-coord labels surfaced in dev mode
Right now agent positions are tile-precise but invisible. Expose a debug overlay that renders the A01-AJ20 coords on hover and the agent's current (zone_id, tx, ty) in a corner readout. Cheap, makes every other bug report ten times more legible. Hide behind ?debug=tiles.
X2 Door state - open/closed flag
Doors are currently permeable-or-removed. Add an open: bool field that defaults to true. Closing a door at runtime (e.g., the cabin_stay scenario locking cabin secondary doors) becomes a one-line state change instead of a regen. No movement logic change - closed doors are just impassable like walls.
X3 Lifeboat capacity per station
The lifeboat row is currently one long room. Sub-divide into 8 stations (port: 4, starboard: 4) with an explicit capacity: 12 on each. The drill scenario then has somewhere to assign agents and we get a queueing signal we can actually plot. No new geometry - the row already spans 27 tiles.
Disposition Ship is unblocked.
Adjust Q1-Q3, Q5-Q8 + X1-X3 this iteration. Q4 parked for v2. Total YAML delta: ~40 lines including off-list adds. No hull or grid changes - coordinate system stays 36x20 @ 28px. Send me the regen output before large_crowd goes live.
